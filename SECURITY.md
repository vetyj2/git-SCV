# Security Policy

Report security issues privately to the project maintainer before public
disclosure.

Git-SCV is a no-exec inspection tool. A report from Git-SCV must not be treated
as proof that a repository is safe to install, build, test, or run.

When reporting a suspected leak, include the Git-SCV version, command, artifact
names, and whether the value appeared in stdout, stderr, JSON, Markdown, or
HTML. Do not include real secrets in the report; use redacted examples.
