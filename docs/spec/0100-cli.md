# 사양 0100 — 명령 입구와 오류 처리

대상 모듈: `src/main.rs`, `src/cli.rs`(0100), `src/errors.rs`(1110)
관련 요구사항: 0021–0025, 0101–0106, 0105, 1110, 9006

이 문서는 구현 지시 사양이다. 구현자는 이 문서에 적힌 그대로 만들고,
적히지 않은 동작을 임의로 추가하지 않는다.

## 1. 명령 문법

```sh
git-scv inspect <repo-path> --out <run-dir>
git-scv snapshot <archive-url> --out <snapshot-dir> --sha256 <hex>
```

- 의존성: `clap` derive 방식.
- 하위 명령은 `inspect`와 `snapshot`을 둔다.
- `<repo-path>`: 위치 인자, 필수.
- `--out <run-dir>`: 옵션 인자, 필수.
- `snapshot`은 원격 스냅샷 기능의 별도 명령이다. 현재 단계에서는
  `--sha256`이 없으면 파일을 만들기 전에 사용자 오류로 거부한다.
- `--sha256` 값은 64자리 hex SHA-256 digest여야 한다.
- SHA-256 계산과 digest 대조는 `src/snapshot.rs`의 순수 함수로 분리한다.
- `snapshot` URL은 `https://`로 시작하는 원격 압축 주소만 허용한다.
- 지원하는 원격 압축 주소 확장자는 `.zip`, `.tar.gz`, `.tgz`다.
- `snapshot` URL은 사용자 정보(userinfo)를 포함할 수 없다.
- URL 검증 오류는 사용자 정보, query, fragment 원문을 복사하지 않는다.
- 다운로드 본문은 초기 50MiB 한도로 메모리에 읽고, 체크섬 불일치 시
  출력 디렉터리를 만들지 않는다.
- 체크섬이 일치하면 `.zip`, `.tar.gz`, `.tgz` 항목을 출력 디렉터리 안에만
  압축 해제한다. `..`, 절대 경로, symlink, hardlink, 특수 파일 항목은
  출력 디렉터리를 만들기 전에 거부한다.
- 압축 해제 대상은 `<snapshot-dir>/source`이고, 기존 검사 산출물은
  `<snapshot-dir>/run`에 쓴다.
- `snapshot --out`도 새 경로 또는 빈 디렉터리여야 한다.

## 2. 도움말 고정 문구

`git-scv --help`와 `git-scv inspect --help` 모두, 본문 마지막 줄에
아래 문장을 한 글자도 바꾸지 않고 넣는다(0106):

```text
git-scv는 대상 저장소의 어떤 명령, 스크립트, 훅도 실행하지 않는다.
```

clap의 `after_help`로 넣는다.

## 4. 입력 검증 (실행 전, 어떤 파일도 만들기 전에 수행)

검증 순서와 오류 메시지는 아래 표와 같다. 첫 실패에서 즉시 중단하고
코드 2로 종료한다. 이 시점에는 출력 디렉터리를 포함해 아무것도 만들지
않는다(0103).

| 번호 | 조건 | stderr 메시지 (한 줄, 그대로) |
| --- | --- | --- |
| U00 | `<repo-path>`가 저장소 URL 형태임 (0023, 0026, 1401) | `오류: 저장소 URL 입력은 아직 지원하지 않는다. 먼저 로컬로 받은 저장소 경로를 지정한다: <경로>` |
| U01 | `<repo-path>`가 존재하지 않음 (0024) | `오류: 검사 대상 경로가 존재하지 않는다: <경로>` |
| U02 | `<repo-path>`가 디렉터리가 아님 (0024) | `오류: 검사 대상 경로가 디렉터리가 아니다: <경로>` |
| U03 | `--out`이 존재하고 디렉터리가 아님 | `오류: 출력 경로가 디렉터리가 아니다: <경로>` |
| U04 | `--out`이 존재하는 디렉터리이고 비어 있지 않음 (9006 확정) | `오류: 출력 디렉터리가 비어 있지 않다: <경로>` |
| U05 | canonicalize한 `--out`이 canonicalize한 `<repo-path>` 내부 | `오류: 출력 디렉터리가 검사 대상 내부에 있다: <경로>` |
| U06 | `snapshot` 명령에 `--sha256`이 없음 (1402) | `오류: snapshot 명령은 --sha256 체크섬이 필요하다.` |
| U07 | `snapshot` 명령의 `--sha256`이 64자리 hex가 아님 (1402) | `오류: snapshot 명령의 --sha256 값은 64자리 hex여야 한다.` |
| U08 | `snapshot --out`이 존재하고 디렉터리가 아님 | `오류: snapshot 출력 경로가 디렉터리가 아니다: <경로>` |
| U09 | `snapshot --out`이 존재하는 디렉터리이고 비어 있지 않음 | `오류: snapshot 출력 디렉터리가 비어 있지 않다: <경로>` |
| U10 | `snapshot` URL이 `https://` 원격 압축 주소가 아님 | `오류: snapshot URL은 https:// 원격 압축 주소여야 한다: <URL>` |
| U11 | `snapshot` URL이 지원 압축 확장자로 끝나지 않음 | `오류: snapshot URL은 .zip, .tar.gz, .tgz 압축 주소여야 한다: <URL>` |
| U12 | `snapshot` URL authority에 사용자 정보가 있음 | `오류: snapshot URL은 사용자 정보를 포함할 수 없다.` |

- U04: "비어 있음"은 항목 수 0을 뜻한다. `.DS_Store` 같은 숨김 파일이
  있어도 비어 있지 않은 것이다. 덮어쓰기는 어떤 경우에도 하지 않는다.
- U00: `https://`, `ssh://`, `git://`, `file://`, `git+ssh://`처럼 URL
  scheme을 가진 값과 `git@host:repo` 형태를 포함한다. 원격 스냅샷 기능
  구현 전까지 URL 입력은 로컬 경로로 해석하지 않는다.
- U00, U10, U11의 `<URL>` 또는 `<경로>` 표시에는 URL 사용자 정보,
  query, fragment 원문을 그대로 넣지 않는다.
- `--sensitive-path` 값은 저장소 상대 경로여야 하며 URL scheme 값이나
  `git@host:repo` 형태는 승인 경로로 해석하지 않는다(0027).
- `redacted-summary`는 `--approve-sensitive-review`와
  `--sensitive-review-ack review-sensitive-candidates`를 모두 요구한다.
- `approved-raw`는 위 1차 승인에 더해 `--approve-sensitive-raw`,
  `--sensitive-raw-ack include-approved-sensitive-raw-in-diagnostic-input`,
  하나 이상의 `--sensitive-path`를 모두 요구한다(1113A).
- U05 비교 시 `--out`이 아직 없으면 존재하는 가장 가까운 부모를
  canonicalize해 비교한다.
- 오류 메시지 뒤에 clap 사용법(usage) 한 블록을 stderr로 출력한다.
- 모든 오류 출력은 stderr 전용이다. stdout에는 오류를 쓰지 않는다.

## 5. errors.rs (1110)

```rust
pub enum ScvError {
    Usage(String),            // 종료 코드 2
    Inspect(String),          // 종료 코드 3, "<단계이름>: <사유>" 형식
    Validation(Vec<String>),  // 종료 코드 4, 실패한 V번호 문자열 목록
}

impl ScvError {
    pub fn exit_code(&self) -> i32;   // 위 표의 매핑 그대로
    pub fn user_message(&self) -> String; // stderr에 그대로 출력할 문장
}
```

- `user_message` 형식:
  - Usage: 4절 표의 문장 그대로.
  - Inspect: `오류: 검사 실패(<단계이름>): <사유>`
  - Validation: `오류: 산출물 검증 실패: V02, V05` 처럼 V번호를 쉼표로 나열.
- 내부 디버그 정보(러스트 오류 체인)는 사용자 메시지에 섞지 않는다.
  필요하면 run.json의 stage error 필드에만 남긴다(0202).

## 6. main.rs

- `main`은 `cli::parse()` → `inspect::run(args)` 호출 →
  `Err(e)`면 `e.user_message()`를 stderr에 출력하고
  `std::process::exit(e.exit_code())`. 그 외 로직 금지.
- 주의: `std::process::exit`는 허용된다. 금지 대상은 프로세스 **생성**
  API(`std::process::Command` 등)다. 무실행 회귀 시험(T09)의 금지 문자열
  목록은 `Command::`, `.spawn(`, `.output(`, `.status(` 로 한다.
  (`std::process::exit`는 이 목록에 걸리지 않는다.)

## 7. 성공 시 stdout

성공(코드 0) 시 stdout에 정확히 두 줄을 출력한다:

```text
검사 완료: <path-privacy-policy 적용 repo 표시>
산출물: <canonicalize된 out-dir>
```

기본 `repo-relative` path privacy에서는 첫 줄이 `<repo-root>`로 출력된다.
그 외 진행 로그는 출력하지 않는다. 에이전트가 stdout을 파싱하지 않고
산출물 파일만 읽는 것이 계약이므로(1203), stdout은 사람 확인용 최소만 둔다.
