# 사양 0400 — 파일 탐색

대상 모듈: `src/walk.rs`(0400)
관련 요구사항: 0401–0406, 신규 0407, 0408, 9007 확정

출력 구조는 사양 0900의 inventory.json 스키마를 따른다.

## 1. 순회 규칙

- `walkdir` 크레이트, `follow_links(false)` 고정(9007 확정).
- 루트 자신은 항목에 넣지 않는다. 같은 깊이에서는
  `sort_by_file_name`(이름 바이트 사전순)으로 순회한다.
- 최종 `entries`와 `skipped` 목록은 경로 바이트 사전순으로 다시 정렬해
  기록한다. 같은 입력이면 항상 같은 바이트가 나와야 한다(시험 T10).
- 한도 없음(9003 확정): 파일 수, 깊이, 크기 한도를 두지 않는다.
  `limits`는 `{"max_files": null, "max_read_bytes_per_file": null}`로
  명시 기록한다(0205).

## 2. 항목 기록 (0401, 0402)

각 항목은 다음을 기록한다:

- `path`: 루트 기준 상대 경로, 구분자는 `/`. 윈도우 경로 변환은 최소
  기능판 범위 밖이지만 구분자 규칙은 코드에 박는다.
- `kind`: `file` | `dir` | `symlink` | `other`.
  `other`는 fifo, 소켓, 장치 파일 등이다. 판정은 `symlink_metadata`
  (lstat) 기준 — 링크를 따라가서 판정하지 않는다.
- `size`: `kind=file`일 때만 바이트 수. `dir`, `symlink`, `other`는 null.
- `ext`: 마지막 확장자를 소문자로, 점 없이. 없으면 null.
  예: `setup.SH` → `"sh"`, `Makefile` → null.
- `symlink_target`: `kind=symlink`일 때 `read_link` 결과 문자열.
  읽기 실패 시 null. 그 외 kind는 null.

## 3. 제외와 건너뜀 (0403–0405, 0407, 0408)

- **`.git` 디렉터리**(루트 직속): entries에 올리지 않고 내부를 순회하지
  않는다. skipped에 `{"path":".git","reason":"excluded-git-dir"}` 한 건을
  기록한다(신규 0408).
- **무시 규칙 미적용**(신규 0407): `.gitignore`, `.ignore` 등 어떤 무시
  규칙도 적용하지 않는다. 악성 저장소는 무시 규칙으로 위험물을 숨길 수
  있다. 숨김 파일과 숨김 디렉터리도 전부 entries에 올린다.
  이 정책은 `policy` 필드에 기록한다(0405).
- **심볼릭 링크**(9007 확정): entries에 `kind:"symlink"`로 올리고,
  동시에 skipped에 `{"path":…,"reason":"symlink"}`를 기록한다.
  내부로 절대 들어가지 않는다.
- **읽기 오류**: 메타데이터를 읽을 수 없는 항목은 entries에 올리지 않고
  skipped에 `reason:"unreadable"`로 기록한 뒤 검사를 계속한다(0403).
  디렉터리 열기에 실패하면 그 디렉터리 자체를 `unreadable`로 한 건
  기록하고 내부는 세지 않는다.

`skipped[].reason`의 허용값은 `symlink` | `unreadable` |
`excluded-git-dir` 세 개뿐이다.

## 4. 집계 (0406)

- `totals.discovered`: 순회 중 만난 모든 항목 수 (entries + skipped).
- `totals.listed`: entries 길이.
- `totals.skipped`: skipped 길이.
- 불변식 `discovered == listed + skipped`는 검증 관문 V03이 검사한다.
- 심볼릭 링크는 entries와 skipped 양쪽에 기록되므로, **집계 시에는
  skipped 쪽으로만 센다**(listed에서 제외). 즉
  `listed = entries 중 kind != symlink 인 항목 수`,
  `discovered = listed + skipped`. 이 규칙은 V03 검사 코드와 시험
  T08이 똑같이 따른다.
